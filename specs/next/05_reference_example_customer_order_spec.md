# Hintergrund-Spec: Referenzbeispiel Auftragsdokumentation fuer Handwerker

## Zweck

Dieses Dokument definiert den fachlichen Referenzfall fuer die neue vereinfachte Richtung.
Es bleibt als Hintergrund fuer die erste Ziel-App nuetzlich, ist aber nicht mehr die fuehrende Roadmap-Spec.

## Fachlicher Zweck

Das Referenzformular bildet einen realistischen Handwerkerauftrag ab:

- Auftrag anlegen
- Auftragsdaten erfassen
- Arbeitsfortschritt dokumentieren
- Nachweise und Bilder beilegen
- Freigabe und Abschluss nachvollziehbar machen

## Typische Felder

Typische Felder sind:

- Auftragsnummer
- Kunde
- Einsatzort
- Ansprechpartner
- Leistungsbeschreibung
- Termin oder Zeitraum
- Status der Ausfuehrung
- Notizen zur Ausfuehrung
- Nachweise / Bilder

## Typische einfache Actions / Freigaben

Typisch fuer den Referenzfall sind:

- Auftrag starten
- Zwischenspeichern
- zur Pruefung einreichen
- freigeben
- ablehnen
- archivieren

## Warum dieses Beispiel geeignet ist

Dieses Beispiel ist geeignet, weil es:

- fachlich nah an realer Dokumentationsarbeit ist
- einfache Formeingaben, Bilder und Notizen kombiniert
- einen nachvollziehbaren Freigabefluss hat
- ohne technische Sonderarchitektur verstaendlich bleibt

## Ziel fuer die neue Richtung

Der Referenzfall soll spaeter zeigen:

- dass das vereinfachte Formularmodell praxisnah bleibt
- dass direkte lokale Bindungen ausreichen
- dass das einfache Layoutmodell genuegt
- dass Arbeits-UI und Review-/Konfigurationssicht sauber getrennt werden koennen

## In dieser Spec bewusst noch nicht festgelegt

- konkrete Feldsyntax
- finale Action-Syntax
- endgueltiges Datenmodell
- konkrete Beispielinstanz im Code
